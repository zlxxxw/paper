# 实验平台与软件条件

## 上位机硬件平台

论文实验的数据处理、模型训练、离线分析和部分仿真可依托高性能上位机完成。当前上位机硬件配置如下：

- 处理器：Intel Core i9-13900K
- 系统内存：64 GB
- GPU：NVIDIA RTX A4000
- GPU 显存：16 GB 专用显存

该平台适合承担 YOLO 模型训练与验证、实验数据回放、轨迹误差统计和可视化分析等任务。车载端则主要承担标线 / 障碍感知、局部运动推算、避让决策和底盘控制。

## 实验室物理模拟平台

实验场地为室内可控光照机场机坪 / 机位物理模拟平台。平台采用 1:10 缩比 A320 机位场景，用于复现实验所需的机位局部作业环境。

平台包含以下要素：

- 1:10 A320 航空器模型
- 机坪地皮与机位地面标线
- 机位红线
- 黄色中线 / 航空器引导线
- 白线斑马线 / 等待区标记
- 1:10 地勤人员模型
- 用于扮演有人保障车辆的车辆模型
- 多辆可控 ROS 小车，用于扮演无人保障车辆

该平台可用于验证车辆在服务车道或等待区启动、驶入机位纵向保障通道、沿红线纵向巡行、切入横向作业通道、靠近近机作业点和异常情况下安全驶离过程中的轨迹控制与避让行为。室内可控光照条件有利于降低环境变化对标线感知和目标检测实验的干扰。

## 无人保障车辆平台

无人保障车辆采用 Yahboom ROSMASTER R2 系列 ROS 小车作为缩比实验载体。小车搭载 Jetson Orin NX 16G 边缘计算单元，可运行 ROS 节点、标线感知、目标检测、深度反投影、局部运动推算、避让决策和底盘控制程序。

车载硬件包括：

- 底盘平台：Yahboom R2 车型 ROS 小车
- 边缘计算单元：Jetson Orin NX 16G
- 车顶相机：DCXGW12030 单目 RGB 相机，可选搭载
- 前向 RGB-D 相机：Astra Pro Plus
- 激光雷达：YDLIDAR 4ROS 单线激光雷达
- 惯性测量单元：IMU
- 轮速里程计：由底盘编码器或车载里程计节点提供

各传感器在系统中的作用如下：

- DCXGW12030 单目 RGB 相机为可选搭载设备，可用于实验视频记录、标线辅助感知或冗余视觉输入，不作为本文 3D 定位前提。
- Astra Pro Plus RGB-D 相机用于动态障碍物检测与深度反投影，获得目标相对距离和空间位置。
- YDLIDAR 4ROS 单线激光雷达用于局部障碍物感知、里程计辅助和安全距离判断。
- IMU 用于姿态估计，并与轮速里程计、激光雷达和局部视觉 / 深度感知信息共同支撑无 GNSS 条件下的局部运动推算。
- 轮速里程计用于纵向巡行距离累计，触发预设转向距离处的横向切入状态切换。

## 软件条件

系统软件以 ROS 为核心组织各功能模块。软件条件可按以下层次描述：

- 操作系统与机器人中间件：Ubuntu / ROS，具体版本以后续实车环境为准。
- 底盘驱动：基于 Yahboom ROSMASTER R2 官方 ROS 工作空间和底盘控制节点进行适配。
- 标线与通道感知：利用机位红线、等待区标记、黄色引导线和服务车道边界提取局部通道约束。
- 目标检测：采用 YOLO 识别地勤人员、有人保障车辆、其他无人保障车辆和静态障碍物。
- 深度反投影：利用 Astra Pro Plus RGB-D 数据将检测框内目标反投影到车辆坐标系或局部通道坐标系。
- 局部运动推算：融合轮速里程计、IMU、激光雷达和局部视觉 / 深度感知信息，在无 GNSS 条件下估计车辆相对运动状态。
- 虚拟通道生成：根据服务车道边界、机位红线、等待区标记、航空器引导线和近机作业点生成纵向主线、横向作业通道、驶入通道和安全撤离通道。
- 轨迹跟踪控制：采用 Pure Pursuit 或 Stanley 控制器完成纵向巡行、横向切入、低速靠近和原路撤离。
- 状态机控制：管理服务车道驶入、等待区启动、纵向巡行、转向切入、横向靠近、作业等待、反向撤离、异常驶离和停车避让等状态。
- 分区避让决策：结合车辆当前区域、目标类别和目标位置，执行纵向受限绕行或横向停车等待策略。
- 数据记录与分析：使用 rosbag、轨迹日志和实验指标统计脚本记录传感器、定位、控制和避让过程。
