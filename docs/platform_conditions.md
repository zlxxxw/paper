# 实验平台与软件条件

## 上位机硬件平台

论文实验的数据处理、模型训练、离线分析和部分仿真可依托高性能上位机完成。当前上位机硬件配置如下：

- 处理器：Intel Core i9-13900K
- 系统内存：64 GB
- GPU：NVIDIA RTX A4000
- GPU 显存：16 GB 专用显存

该平台适合承担 YOLO 模型训练与验证、风险分级模型调参、实验数据回放、风险事件统计和结果可视化等任务。车载端则主要承担实时目标检测、深度反投影、局部空间映射、风险等级判断、避让决策和底盘控制。

## 实验室物理模拟平台

实验场地为室内可控光照机场机坪 / 机位物理模拟平台。平台采用 1:10 缩比 A320 机位场景，用于复现近机保障过程中人员、车辆和障碍物在结构化通道内交互的风险场景。

平台包含以下要素：

- 1:10 A320 航空器模型
- 机坪地皮与机位地面标线
- 机位红线
- 黄色中线 / 航空器引导线
- 白线斑马线 / 等待区标记
- 1:10 地勤人员模型
- 用于扮演有人保障车辆的车辆模型
- 多辆可控 ROS 小车，用于扮演无人保障车辆

该平台可用于验证无人 GSE 在纵向通行通道、横向近机作业通道和近机作业区内遇到不同风险对象时的分类、分级和避让行为。实验场景可覆盖人员穿越、车辆占道、其他无人 GSE 交互、静态障碍物侵入通道、远离航空器侧绕行空间充足或不足等典型情况。

## 无人保障车辆平台

无人保障车辆采用 Yahboom ROSMASTER R2 系列 ROS 小车作为缩比实验载体。小车搭载 Jetson Orin NX 16G 边缘计算单元，可运行 ROS 节点、目标检测、深度反投影、激光雷达处理、风险分级、状态机决策和底盘控制程序。

车载硬件包括：

- 底盘平台：Yahboom R2 车型 ROS 小车
- 边缘计算单元：Jetson Orin NX 16G
- 车顶相机：DCXGW12030 单目 RGB 相机，可选搭载
- 前向 RGB-D 相机：Astra Pro Plus
- 激光雷达：YDLIDAR 4ROS 单线激光雷达
- 惯性测量单元：IMU
- 轮速里程计：由底盘编码器或车载里程计节点提供

各传感器在系统中的作用如下：

- DCXGW12030 单目 RGB 相机为可选搭载设备，可用于实验视频记录、标线辅助感知或冗余视觉输入，不作为本文 3D 定位前提。
- Astra Pro Plus RGB-D 相机用于风险对象检测与深度反投影，获得目标相对距离、横向偏移和空间位置。
- YDLIDAR 4ROS 单线激光雷达用于近距离障碍物感知、通道占用确认和安全距离判断。
- IMU 用于姿态估计，并与轮速里程计共同支持短时相对运动推算和 TTC 估计。
- 轮速里程计用于估计车辆自身速度、相对位移和通道内短时运动状态，不再作为论文中定点转向控制的核心依据。

## 软件条件

系统软件以 ROS 为核心组织各功能模块。软件条件可按以下层次描述：

- 操作系统与机器人中间件：Ubuntu / ROS，具体版本以后续实车环境为准。
- 底盘驱动：基于 Yahboom ROSMASTER R2 官方 ROS 工作空间和底盘控制节点进行适配。
- 结构化通道区域建模：根据服务车道边界、机位红线、等待区标记、航空器引导线和近机作业点生成纵向通行通道、横向近机作业通道、近机作业区和航空器侧安全边界。
- 风险对象检测：采用 YOLO 识别地勤人员、有人保障车辆、其他无人保障车辆和静态障碍物。
- 深度反投影与空间映射：利用 Astra Pro Plus RGB-D 数据将检测框内目标反投影到车辆坐标系或局部通道坐标系，并结合激光雷达距离信息判断占道关系。
- 局部运动推算：融合轮速里程计、IMU 和局部感知结果，估计车辆速度、相对位移、目标相对速度和 TTC 等风险特征。
- 风险量化与分级：根据目标类别、通道区域、距离 / TTC、横向占道状态和运动趋势生成风险等级。
- 分区避让决策：结合风险等级和通道区域，输出正常通行、减速观察、远离航空器侧受限绕行、停车等待或紧急制动。
- 受控执行：采用底盘控制、Pure Pursuit 或 Stanley 等低层控制器执行决策层给出的动作，控制器不是本文核心创新。
- 数据记录与分析：使用 rosbag、实验日志和指标统计脚本记录传感器数据、风险等级、决策动作、响应时间和安全距离。
