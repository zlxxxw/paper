# 研究基础

## 研究题目

机场机位无人保障车辆结构化通道行驶与分区避让方法

## 问题边界

本文研究的是机位局部作业场景下无人保障车辆在无导航地图或无 GNSS 信号条件下的结构化行驶与避让问题。研究重点不是跨机位、跨道路、复杂路网中的全局路径规划，而是在机位及邻近服务车道范围内，利用既有机位标线和无人保障作业通道框架完成可约束、可解释的局部通行。

研究包含两类典型任务：一类是车辆从服务车道或等待区自动驶入对应近机作业区域；另一类是在保障过程中遇到无图、无信号或其他异常情况时，车辆沿结构化安全通道驶离机位，以保障航空器和人员安全。车辆运行范围主要包括：

- 服务车道与机位入口区域
- 等待区
- 机位红线附近的纵向巡行区域
- 由红线切入近机方向的横向作业通道
- 近机作业点附近的低速靠近区域
- 异常情况下的安全撤离通道

## 场景假设

1. 车辆具有基础底盘控制能力，可接收速度、转角或轨迹跟踪指令。
2. 车辆具备轮速里程计，可累计纵向巡行距离。
3. 机位关键标线和作业点在系统中可被标定，包括机位红线、等待区白线、航空器引导黄线和近机作业点。
4. YOLO 模型可识别人员、有人保障车辆、其他无人车和静态障碍物。
5. 系统不依赖机场全局导航地图或 GNSS 定位结果，主要依托机位标线、车辆里程计、IMU、雷达、RGB-D 感知和任务先验完成局部通行。

## 实验条件基础

本文实验依托室内可控光照的 1:10 机场机坪 / 机位物理模拟平台。平台包含 A320 航空器模型、机位标线、黄色中线 / 航空器引导线、白线斑马线 / 等待区标记、地勤人员模型、有人保障车辆模型和多辆可控 ROS 小车。

无人保障车辆采用 Yahboom R2 车型 ROS 小车，搭载 Jetson Orin NX 16G 边缘计算单元。车顶 DCXGW12030 单目 RGB 相机为可选搭载设备，可用于实验记录、标线辅助感知或冗余视觉输入，但不作为本文 3D 定位前提；车体前方 Astra Pro Plus RGB-D 相机用于动态障碍物检测与深度反投影；YDLIDAR 4ROS 单线激光雷达、IMU 和轮速里程计用于近距离障碍约束、姿态估计和无 GNSS 条件下的局部运动推算。

上位机平台配置为 Intel Core i9-13900K 处理器、64 GB 系统内存和搭载 16 GB 专用显存的 NVIDIA RTX A4000 GPU，主要用于 YOLO 模型训练、实验数据分析、轨迹统计和结果可视化。

## 技术路线

车辆从服务车道或等待区启动后，首先根据机位标线、通道先验和局部感知进入纵向保障通道，并沿机位红线或虚拟纵向主线巡行。系统通过轮速里程计累计车辆沿通道方向的行驶距离，并结合标线、区域边界或预设距离判断转向区域。当满足转向条件后，车辆由纵向巡行状态切换至横向转向状态。

横向切入阶段采用预定义轨迹，并通过 Pure Pursuit 或 Stanley 控制器实现轨迹跟踪。车辆完成转向后进入横向作业通道，以低速中心线跟踪方式靠近近机作业点。完成作业后，车辆按照原路径或预设安全通道撤离；若保障过程中发生无图、无信号或异常占道等情况，系统优先切换至安全驶离策略，使车辆离开航空器敏感区域。

运行过程中，YOLO 持续进行目标识别。系统根据车辆当前区域、目标类型和目标相对位置进行避让决策。纵向主线区域允许车辆向远离航空器一侧进行受限绕行；横向作业通道由于靠近航空器，禁止绕行，车辆停车等待目标清空后继续执行任务。

## 无图无 GNSS 条件下的结构化通道生成思想

本文不在真实机位地面额外绘制纵向主线和横向通道，而是在 ROS 控制系统中生成虚拟通道。虚拟通道由真实机位标线和作业点锚定：

- 服务车道边界和机位入口用于约束驶入方向。
- 机位红线用于生成纵向主线。
- 等待区白线用于确定固定起点和初始姿态。
- 航空器引导黄线用于约束近机方向和航空器侧区域。
- 近机作业点用于确定横向通道终点和低速靠近目标。

这种方式保留机场机位已有标线体系，不增加额外物理标识，也不依赖完整导航地图或 GNSS 坐标，使无人车控制系统在局部范围内具备明确的轨迹参考和区域语义。

## 分区避让规则

车辆运行区域被划分为纵向主线区域和横向作业通道区域。

纵向主线区域：

- 允许车辆在安全边界内绕行。
- 绕行方向优先选择远离航空器一侧。
- 对人员和动态车辆保持更高安全距离。
- 若障碍物占用过大或绕行空间不足，车辆停车等待。

横向作业通道区域：

- 禁止绕行。
- 检测到人员、车辆或静态障碍物占据通道时停车等待。
- 障碍物清空后继续沿中心线低速靠近或撤离。

## 预期贡献

1. 将机场机位局部无人保障任务从开放导航问题转化为无图无 GNSS 条件下的结构化通道行驶问题。
2. 利用既有机位标线生成控制系统可用的虚拟通道，降低对全局导航地图、GNSS 和场景改造的依赖。
3. 将目标识别结果与区域语义结合，形成适合近机作业安全约束的分区避让策略。
