# 实验设计草案

## 实验目标

实验应验证三个层面的有效性：

1. 结构化任务链是否能够稳定完成等待区启动、纵向巡行、横向切入、近机靠近和原路撤离。
2. 虚拟通道是否能在不额外绘制物理通道的条件下，为车辆提供稳定轨迹参考。
3. 分区避让策略是否能根据纵向主线和横向作业通道的不同安全约束，执行符合近机保障要求的避让行为。

## 实验平台

实验依托室内可控光照机场机坪 / 机位物理模拟平台和 ROS 小车完成。上位机硬件配置为 i9-13900K 处理器、64 GB 系统内存和 NVIDIA RTX A4000 GPU，其中 RTX A4000 具有 16 GB 专用显存，可用于 YOLO 模型训练、实验数据分析和轨迹可视化。

物理场景采用 1:10 A320 机位模拟平台，包括地皮、机位标线、黄色中线 / 航空器引导线、白线斑马线 / 等待区标记、地勤人员模型、有人保障车辆模型和多辆可控 ROS 小车。

无人保障车辆采用 Yahboom R2 车型 ROS 小车，搭载 Jetson Orin NX 16G 边缘计算单元。车载传感器包括：

- DCXGW12030 单目 RGB 相机：可选安装于车顶，用于实验视频记录、标线辅助感知或冗余视觉输入，不作为 3D 定位前提。
- Astra Pro Plus RGB-D 相机：安装于车体前方，用于动态障碍物检测与深度反投影。
- YDLIDAR 4ROS 单线激光雷达：用于局部障碍物感知、距离约束和融合定位辅助。
- IMU：用于姿态估计。
- 轮速里程计：用于纵向巡行距离累计、状态切换触发和里程计推算。

ROS 控制系统包括：

- 无人车底盘与底层控制接口
- 标线与局部通道感知模块
- YOLO 目标检测模块
- RGB-D 深度反投影模块
- 轮速里程计、IMU、激光雷达和局部感知辅助的运动推算模块
- 虚拟通道生成模块
- 路径跟踪控制模块
- 状态机与避让决策模块
- rosbag 数据记录与离线分析模块

Yahboom ROS 小车官方资料和开源工作空间可作为底盘控制、传感器接入和 ROS 节点组织的参考，包括 [zlxxxw/yahboomcar_ws](https://github.com/zlxxxw/yahboomcar_ws.git) 和 [Yahboom ROSMASTER-R2 学习资料](https://www.yahboom.com/study/ROSMASTER-R2#xuanzhon_31)。

## 场景布置

实验场景应包含以下元素：

- 等待区固定起点
- 服务车道与机位入口
- 机位红线或等效纵向标线
- 黄色中线 / 航空器引导线或等效航空器侧约束线
- 白线斑马线 / 等待区标记
- 横向作业通道虚拟中心线
- 近机作业点
- 地勤人员模型、有人保障车辆模型、其他 ROS 小车或静态障碍物的测试位置

## 实验一：完整任务链运行实验

目的：验证车辆能按固定任务链完成无导航地图 / 无 GNSS 条件下的结构化行驶。

流程：

1. 车辆从服务车道或等待区固定起点启动。
2. 系统不调用全局导航地图和 GNSS 定位结果。
3. 车辆根据标线、局部通道约束和里程计信息进入红线纵向巡行。
4. 轮速里程计累计距离达到预设转向距离或车辆进入预设转向区域。
5. 车辆切换至横向转向模式。
6. 车辆进入横向作业通道并低速靠近近机作业点。
7. 模拟作业完成信号或异常驶离信号。
8. 车辆沿原路径或安全撤离通道驶离机位。

评价指标：

- 任务完成率
- 总任务耗时
- 纵向巡行横向误差
- 横向通道中心线跟踪误差
- 转向切入最大偏差
- 停止点与近机作业点距离误差
- 异常驶离成功率

## 实验二：虚拟通道跟踪实验

目的：验证真实标线锚定的虚拟通道可在无导航地图 / 无 GNSS 条件下为车辆提供稳定参考。

变量：

- 不同预设转向距离
- 不同横向通道宽度
- 不同近机作业点位置
- 不同车辆初始横向偏差
- 从服务车道进入机位时的不同初始姿态

评价指标：

- 平均路径跟踪误差
- 最大路径跟踪误差
- 转向切入成功率
- 是否越出虚拟通道边界
- 控制指令平滑性
- 服务车道至机位入口衔接成功率

## 实验三：纵向主线受限绕行实验

目的：验证纵向主线区域允许向远离航空器侧绕行的策略。

障碍物设置：

- 人员位于纵向主线前方
- 有人保障车辆占据部分纵向通道
- 静态障碍物位于红线附近

预期行为：

- 系统识别目标类型。
- 判断车辆处于纵向主线区域。
- 若远离航空器侧具有足够空间，则生成受限绕行轨迹。
- 若绕行空间不足，则停车等待。

评价指标：

- 识别成功率
- 绕行成功率
- 最小安全距离
- 绕行横向偏移量
- 停车等待触发准确率
- 是否出现向航空器侧绕行行为

## 实验四：横向通道禁绕停车实验

目的：验证横向通道靠近航空器时禁止绕行的安全策略。

障碍物设置：

- 人员进入横向作业通道
- 车辆横穿横向作业通道
- 静态障碍物占据横向通道中心线

预期行为：

- 系统识别目标并确认车辆处于横向作业通道区域。
- 车辆停车等待，不生成绕行轨迹。
- 目标离开或障碍清除后，车辆继续低速中心线跟踪。

评价指标：

- 停车响应时间
- 停车距离
- 误绕行次数
- 恢复行驶时间
- 横向通道内安全距离

## 对比方案

建议设置以下对比方案：

1. 无分区避让策略：所有区域采用同一避障规则。
2. 横向通道允许绕行策略：用于证明近机区域禁绕规则的安全必要性。
3. 无虚拟通道约束策略：仅依赖局部避障或普通路径跟踪。
4. 固定停车策略：遇到任何障碍物均停车，不执行纵向受限绕行。

## 风险与补充说明

如果实车实验条件不足，可以采用仿真或缩比实验作为主要验证方式，但论文中需要清楚说明实验平台与真实机位之间的对应关系。对于 YOLO 识别精度，建议单独给出测试集或现场样本下的检测结果，避免避让实验只验证控制策略而没有体现感知模块可靠性。
